Customer & Loan Profiling Agent

Credit underwriting for Secured loans is highly dependent on Underwriter's vintage and hence inconsistent.

AI Agent Fintech LLM

The Problem

  • Underwriters spend significant time manually analyzing fragmented data across bureau, banking, property, and verification reports.
  • Insights are dependent on individual experience, leading to inconsistency in credit assessment and documentation.
  • There is no standardized, real-time way to convert raw loan data into structured, decision-ready narratives.

Solution Highlights

  • Built an AI-powered underwriting co-pilot that generates structured customer & loan profiles in seconds.
  • Converts multi-source loan data into consistent, narrative credit summaries with strengths, weaknesses, and risks.
  • Designed as an event-driven, embeddable agent that can plug into any LOS workflow.

Metrics That Will Move

How success gets measured.

Detailed Solution

  • Built a FastAPI-based microservice exposing an endpoint (/generate-profile) that accepts structured loan application data (customer, bureau, banking, property, FI, fraud, PD).
  • Designed a Pydantic-based schema layer to validate and normalize incoming LOS data before passing it to the AI model.
  • Implemented an AI agent service layer that constructs a contextual prompt and interacts with the LLM to generate structured underwriting summaries.
  • Engineered a system prompt tailored to underwriting workflows, ensuring outputs are factual, structured, and aligned with credit memo standards (400 characters per section).
  • Enforced structured outputs via schema-based parsing, eliminating JSON parsing failures and ensuring deterministic response format.
  • Constrained the model using a strict system prompt + structured output schema + pre-validated input data, ensuring reliability; the model is used for interpretation, not computation (all financial metrics are pre-computed).
  • Integrated token usage tracking (input_tokens, output_tokens, total_tokens) to monitor cost and optimize prompt efficiency (~1300–1800 tokens per request).
  • Designed output into 5 standardized sections: Customer Profile, Property Profile, Case Strengths, Case Weaknesses, Potential Risks — mirroring real underwriting decision formats.
  • Built extensibility for external data ingestion via APIs and services: bureau providers, banking analyzers, property tech/legal vendors, fraud detection systems.
  • Architected as an independent, event-driven agent that can be triggered by LOS events (case creation, stage movement, manual trigger, or webhook), enabling seamless embedding across workflows.
  • Uses agentic LangGraph architecture (instead of prompt based data ingestion), where the LLM dynamically fetches required data via tool calls (using case_id) instead of receiving full payloads, enabling efficient multi-step reasoning and controlled execution.

Prompt Used

You are an experienced retail credit underwriter responsible for evaluating loan applications and preparing structured case summaries.

Your task is to analyze structured loan application data and generate a concise, professional underwriting summary.

Rules:
1. Use only the data provided in the input. Do not assume or invent any information.
2. Maintain a formal, factual, and credit-oriented tone similar to internal underwriting notes.
3. Avoid generic statements; ensure each point is grounded in the data.
4. Each section must be descriptive, human-readable, and within 400 characters.
5. Do not repeat the same information across multiple sections.
6. Mention key financial metrics where relevant (e.g., bureau score, FOIR, LTV, income, DPD).
7. Highlight repayment capacity, credit behavior, and collateral quality clearly.
8. If any data is missing, do not infer or fill gaps — simply omit it.
9. Ensure the output is structured and suitable for direct use in a credit memo.

Output Format (strict JSON):
{
  "customer_profile": "",
  "property_profile": "",
  "case_strengths": "",
  "case_weaknesses": "",
  "potential_risks": ""
}
